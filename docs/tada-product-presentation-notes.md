# TaDa! Product Presentation Notes

## Slide 1
Open with the promise: TaDa! is presentation mode for HTML. It is especially useful now that AI tools create polished HTML artifacts.

## Slide 2
Frame the shift. Claude, Codex, Gemini, and ChatGPT can produce useful HTML, but the handoff into a client presentation is still clumsy.

## Slide 3
Define the product boundary. TaDa! is not an editor or generator; it is the focused presentation layer for HTML.

## Slide 4
Explain the normalizer. TaDa! looks for common slide boundaries first, then falls back when generated HTML is less structured.

## Slide 5
Use Ely as the concrete proof point. This is a real sample deck loaded into the app, not a theoretical mock.

## Slide 6
Talk through the presenter workflow: thumbnails, arrow keys, side panel, presenter notes, fullscreen, and escape-to-return.

## Slide 7
Emphasize fidelity. The product goal is to keep CSS, JavaScript, canvas, animation, and generated effects running.

## Slide 8
Introduce standalone show export. This is the PowerPoint-style handoff: open the file and it is already in presentation mode.

## Slide 9
Position production readiness: tests, Mac build path, Windows packaging target, and update plumbing.

## Slide 10
Close on the core phrase: TaDa! is presentation mode for HTML.
